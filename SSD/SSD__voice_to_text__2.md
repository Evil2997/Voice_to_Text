# 🧱 ТЕКУЩАЯ СТРУКТУРА (смысловая)

## 🧠 Domain

* models
* exceptions
* run_logic
* (должен владеть портами: RunRepository)

Domain:

* знает правила
* не знает SQLite
* не знает Whisper
* не знает ffmpeg

---

## 🔧 Infrastructure

* sqlite (schema + repo)
* whisper (engine)
* audio (ffmpeg / yt-dlp / media)
* settings (конфиг)

Infrastructure:

* реализует контракты
* ничего не знает о бизнес-сценариях

---

## 🎛 Application

* prod_service
* bench_service

Application:

* собирает зависимости
* оркестрирует
* не реализует низкоуровневые детали

---

## 🛎 CLI

* entrypoint
* запуск режима

CLI:

* только точка входа

---

# 🧠 КЛЮЧЕВЫЕ ИДЕИ, КОТОРЫЕ ТЫ ОСВОИЛ

1. Контракт принадлежит потребителю (Domain).
2. Domain не зависит от Infrastructure.
3. Application связывает домен и адаптеры.
4. Infrastructure — просто инструменты.
5. Settings — это конфигурация, не CLI.
6. Persistence — это заменяемый слой.
7. Engine lifecycle важен для производительности.
